Workstream 1: Standalone mem0 Memory Service
                                                                                                                                                                                                                                                                                               
     Context
                                                                                                                                                                                                                                                                                               
     Trybo's orchestration engine needs long-term memory as ambient infrastructure — not a skill. Memory retrieves user context before every GID/Planner call and stores learned facts after every graph completion. This PR builds the standalone mem0 service with zero integration        
     wiring (that comes after the co-worker's agentic MVP PR merges).

     We use self-hosted mem0ai v1.0.4 with Supabase/pgvector backend and OpenAI embeddings (reusing existing OPENAI_API_KEY). The service is feature-flagged (MEM0_ENABLED=False by default) so it can ship dark.

     3. app/services/memory/mem0_service.py

     Core service class following existing singleton pattern (app/services/llm_service.py):

     class MemoryService:
         _memory: Memory | None  (lazy-initialized)
         _initialized: bool

         _ensure_initialized() -> bool
             - Returns immediately if MEM0_ENABLED=False (no mem0 import)
             - Returns immediately if SUPABASE_DB_URL not set (log warning)
             - Imports mem0.Memory, builds config from settings, creates instance
             - Catches all exceptions → logs error, stays disabled

         async retrieve_context(user_id, query, limit=10) -> MemoryContext
             - Called before GID/Planner
             - asyncio.to_thread(self._memory.search, ...)
             - Never raises — returns empty MemoryContext on failure

         async store_interaction(user_id, content, metadata=None) -> None
             - Called after graph completion
             - asyncio.to_thread(self._memory.add, ...)
             - Fire-and-forget: failures logged, never raised

         async search(user_id, query, limit=10) -> list[MemoryItem]
         async get_all(user_id) -> list[MemoryItem]
         async delete(memory_id) -> None
         async delete_all(user_id) -> None

         _parse_search_results(raw) -> list[MemoryItem]  (static)
         _parse_get_all_results(raw) -> list[MemoryItem]  (static)

     _memory_service: Optional[MemoryService] = None

     def get_memory_service() -> MemoryService:   # lazy singleton

     Key design:
     - All sync mem0 calls wrapped in asyncio.to_thread()
     - Never-crash contract: memory failures log errors but never propagate exceptions
     - Lazy init: mem0 only imported on first actual use
     - mem0 config uses gpt-4o-mini for extraction LLM, text-embedding-3-small for embeddings

     mem0 auto-creates its tables (memories, memory_histories) in the public schema on first use. No application migration needed.
